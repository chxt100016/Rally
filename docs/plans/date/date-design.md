## 一、首次进入引导（Onboarding）

新用户注册后弹出底部半屏引导弹窗（[ui-onboarding-modal.html](ui-onboarding-modal.html)），分 5 步完善资料：

1. 头像 / 昵称确认（来自微信，可修改）
2. 性别选择（男 / 女 / 不透露）+ 生日填写（选填）
3. NTRP 自评（滚轮选择，1.5~7.0）
4. 个人简介（选填，最多 100 字）
5. 打球视频上传（**必传**，30 秒内；上传后主按钮激活）

用户可点击「先逛逛，稍后完善」跳过，但视频未上传前主按钮保持禁用。

## 二、系统概览

网球约球系统，面向个人网球选手，核心价值是「快速找到合适的球友约球」。

**底部导航：**
- `发现约球`（首页）
- `+`（悬浮按钮，点击弹出发布弹窗）
- `我的`

### 2.3 他人评价

约球结束后，参与者可对同场**除自己外的每位球友**分别评价，每对用户每场约球只能评价一次（最多评价 3 人）。

**评价内容（两部分）：**

① NTRP 投票（三元，必选）

> 「你觉得对方水平比他自评的怎么样？」
> 👆 高了 / 差不多 / 👇 低了

② 个性化标签（多选，选填，每人独立选择）

**预设标签（MVP）：**
正手稳、反手稳、发球好、底线稳、网前好、移动快、守时、友善、爽约（负向）

**比分记录（可选功能）**

约球结束后，任意参与者可选择记录本场比分。记录流程：

1. 创建盘数（可添加多盘）
2. 每盘选择赛制：`4局制` / `6局制` / `抢N分`
3. 选择双方阵容（多人约球时需指定哪些人在哪侧）
4. 输入每盘比分（具体输入交互待讨论）

- 比分记录与评价相互独立，不强制关联。
- 每一次约球，当前盘数-对阵双方球员id相关的比分应该是唯一的。 需要通过本地版本判断，是否能更新成功， 防止多个人同时修改互相覆盖。

### 2.4 用户画像体系

用两个维度对外描述一个球员：**自评 NTRP** + **球友评级**。

```
对外展示    自评 NTRP  +  球友评级（S / A / B / C）
                               ↓ 点击展开
                  信誉分 · 可信度 · 校准度（三项子分）
后台隐藏    真实 ELO 分数（仅用于撮合，不对用户暴露）
```

#### 球友评级（Buddy Rating）

对外只展示等级字母，不展示总分数值。

**总分计算：**

```
总分（0–100）= 信誉分 × 50% + 可信度 × 30% + 校准度 × 20%
```

| 等级 | 分数区间 | 含义 |
|------|---------|------|
| S | 90–100 | 高度可信 |
| A | 75–89 | 可靠球友 |
| B | 55–74 | 基本可信 |
| C | < 55 | 数据不足或存在问题 |

**新用户初始状态：** 注册后默认展示 **A（新人）**，标注「新人」角标，直到收到 3 次及以上他人评价后切换为计算值。

**展示位置：**
- 「我的」页面：球友评级卡片，可点击展开三项子分
- 约球详情页发布者卡片：显示等级字母，可展开查看子分明细
- 约球列表：不展示

---

#### 分数生命周期

**触发时机：** 每次约球状态变为 `finished` 后，系统异步重算参与者的三项子分和总分。

**信誉分生命周期：**
- 初始值：100
- 事件驱动更新（见下方子维度一）
- 无时间衰减，下限 0
- 核查期内：冻结（不清零，不更新）

**可信度生命周期：**
- 初始值：0（新用户无约球记录）
- 约球完成次数取**滚动 90 天**窗口，超出窗口的记录自动失效
- 视频上传分值永久有效（不随时间衰减）
- UTR 绑定分值永久有效
- 核查期内：冻结

**校准度生命周期：**
- 初始值：80（数据不足基准分）
- 基于**全量历史投票**计算偏差比例（不设时间窗口，防止刷票后重置）
- 每次新增投票后重算
- 核查期内：冻结

**ELO 生命周期：**
- 初始值：由自评 NTRP 换算（可信度低时权重 70%）
- 每次约球结束后根据比分和对手水平更新
- 无衰减，持续累积

**核查期触发与解除：**
- 触发：自评修改幅度 ≥ 0.5 级
- 期间：三项子分冻结，对外显示「⚠ 近期更新了自评」
- 解除：完成 2 场约球且收到他人投票后自动解除，分数恢复正常计算

---

#### 子维度一：信誉分（权重 50%，满分 100）

默认 100 分，行为驱动扣加分：

| 行为 | 分值变化 |
|------|---------|
| 被标记「准时」 | +2，上限 +10 |
| 连续 5 次准时 | 额外 +5 |
| 被标记「迟到」 | −10 |
| 被标记「爽约」 | −25 |

下限 0，不会为负。准时/迟到标记由约球结束后参与者互评产生。

---

#### 子维度二：可信度（权重 30%，满分 100）

| 来源 | 分值 |
|------|------|
| 近 90 天完成约球（每次 +8，上限 48） | 0–48 |
| UTR 账号绑定 | +25 |
| 上传打球视频（每个 +9，上限 27） | 0–27 |

> 取近 90 天而非累计，防止老用户躺分。

---

#### 子维度三：校准度（权重 20%，满分 100）

基于他人对自评的三元投票（高了 / 差不多 / 低了）：

| 状态 | 分数 | 说明 |
|------|------|------|
| 票数 < 3 | 80 | 数据不足，给中性基准分 |
| 偏差 < 20% | 100 | 自评准确 |
| 偏差 20–50%，偏低（谦虚） | 75 | 实际比自评强，轻微扣分 |
| 偏差 20–50%，偏高（虚报） | 50 | 实际比自评弱，重扣 |
| 偏差 > 50%，偏低 | 55 | 明显谦虚，建议更新自评 |
| 偏差 > 50%，偏高 | 20 | 明显虚报，重扣 |

偏高惩罚重于偏低，因为虚报水平对其他球友伤害更大。

**系统建议：** 偏差强度 > 20% 时，推送「根据 N 位球友反馈，建议将自评从 X 调整为 Y」。

---

#### 真实 ELO（后台计算，不对用户展示）

用于撮合算法，不暴露具体数值。

**数据来源权重（随可信度动态调整）：**

| 数据源 | 可信度低（<30分） | 可信度中（30–80分） | 可信度高（>80分） |
|--------|-----------------|-------------------|-----------------|
| 自评 NTRP | 70% | 30% | 5% |
| 他人投票方向 | 20% | 40% | 35% |
| 战绩（ELO 压力） | 0% | 20% | 40% |
| UTR 换算（如绑定） | 10% | 10% | 20% |

**战绩影响：**
- 赢了自评更高的人 → 真实 ELO 上调压力
- 输给自评更低的人 → 真实 ELO 下调压力
- 比分悬殊系数：6-0 > 6-4 > 7-6，悬殊越大调整幅度越大
- 约球类型权重：竞技（1.0）/ 练习（0.4）/ 陪练（0）

**自评修改限制：**

| 可修改幅度上限 | 冷却期 |
|--------------|--------|
| 可信度 < 30：±1.0 级 | 30 天 |
| 可信度 30–80：±0.5 级 | 60 天 |
| 可信度 > 80：±0.5 级 | 90 天 |

修改后若幅度 ≥ 0.5 级，进入「核查期」：可信度积分冻结（不清零），对其他用户显示「⚠ 近期更新了自评」，完成 2 场约球并收到他人投票后解除。

下调自评的冷却期减半（主动降级视为诚实行为）。

### 2.5 球员主页

其他用户可查看的公开主页（[ui-player-profile.html](ui-player-profile.html)），从约球详情页发布者卡片「查看主页 ›」进入。

**展示信息：**
- 基本信息：头像、昵称、NTRP 自评、城市、加入时间

**球友评价标签：**
- 展示他人给出的所有标签及累计次数

**打球视频：**
- 展示用户上传的打球视频（最多 5 个）
- 可播放，显示时长

**最近约球：**
- 展示该用户最近参与的约球记录（标题、状态、日期、场地、类型）

### 3.2 发布约球

**交互：** 点击底部 `+` 按钮，底部弹出半屏弹窗

#### 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 否 | 选填；不填时自动生成：`{类型} · {星期} {时间} · {场地简称}` |
| `match_type` | enum | 是 | `single`（单打）/ `double`（双打）/ `multi`（多人） |
| `max_players` | int | 是 | 人数上限，最小 2，最大 10 |
| `date` | date | 是 | 约球日期 |
| `start_time` | time | 是 | 开始时间 |
| `duration` | float | 是 | 时长，滚轮选择：0.5 / 1.0 / 1.5 / 2.0 / 2.5 / 3.0（小时） |
| `venue_name` | string | 是 | 场地名称，来自地图选点 POI |
| `venue_address` | string | 是 | 详细地址 |
| `venue_lat` | float | 是 | 纬度 |
| `venue_lng` | float | 是 | 经度 |
| `level_mode` | enum | 否 | `range`（范围）/ `exact`（指定）/ `above`（以上）/ `below`（以下） |
| `level_min` | float | 否 | NTRP 下限，步长 0.5，范围 1.5~7.0 |
| `level_max` | float | 否 | NTRP 上限，步长 0.5，范围 1.5~7.0 |
| `gender_limit` | enum | 是 | `any`（不限）/ `male`（仅男）/ `female`（仅女），默认 `any` |
| `join_mode` | enum | 是 | `direct`（直接加入）/ `approval`（需审核） |
| `min_credibility` | enum | 是 | `none` / `junior` / `mid` / `senior`，默认 `none` |
| `cost_items` | array | 是 | 费用项列表，见下方结构 |

**cost_items 结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 费用项名称，如「球场费」「球费」 |
| `total_amount` | int | 总费用（分），前端展示元 |

**费用分摊计算：**
`每人分摊 = sum(cost_items.total_amount) ÷ max_players`，人数变化时实时重算，满员后锁定。

**NTRP 选择器交互：**
- 平时显示文本框（如 `3.5 ~ 4.5`）
- 点击数值框触发底部滚轮选择器
- 滚轮可选值：1.5 / 2.0 / 2.5 / 3.0 / 3.5 / 4.0 / 4.5 / 5.0 / 5.5 / 6.0 / 6.5 / 7.0

### 3.3 约球详情页

**页面结构（从上到下）：**

1. **顶部标题区**（蓝色背景）
   - 标题
   - 状态标签：水平匹配状态 / 报名模式 / 招募状态

2. **信息模块**（白色卡片）
   - 2×2 网格：类型 / 人数 / 时间（含时长）/ 水平要求
   - 场地单独一行：场地名 + 详细地址 + 「查看地图 ›」

3. **发布者卡片**
   - 头像 / 昵称 / NTRP / 活跃度星级
   - 「查看主页 ›」入口

4. **参与者卡片**
   - 头像列表（已加入人员）
   - 空位用虚线圆圈占位

5. **AA 费用卡片**
   - 各费用项明细
   - 每人分摊 = 总费用 ÷ 人数

6. **底部操作区（按视角区分）**

#### 访客视角（非创建人）

| 状态 | 按钮 |
|------|------|
| 招募中 + 未报名 + `direct` | 「立即报名」 |
| 招募中 + 未报名 + `approval` | 「申请加入」 |
| 已报名 / 待审核 | 「已报名」（不可点击） |
| 已满员 | 「已满员」（不可点击） |

#### 创建人视角

底部显示「管理约球」按钮，进入管理页，包含：

- **编辑信息**：可修改时间、场地、水平要求、费用等（满员后不可修改人数上限）
- **关闭约球**：手动将状态改为 `closed`
- **审批列表**（仅 `join_mode=approval` 时显示）：
  - 展示所有 `pending` 状态的申请，显示申请人头像 / 昵称 / NTRP / 活跃度
  - 操作：「通过」/ 「拒绝」
  - 通过后 `current_players +1`，达到 `max_players` 时状态自动变 `full`

## 六、功能边界（MVP 范围）

**包含：**

_用户与账号_
- 球员信息完善（自评 NTRP、视频上传，最多 5 个）
- 自评修改限制与核查期

_评价与评级_
- 他人评价：多人支持（最多评价同场 3 人），NTRP 三元投票 + 个性化标签
- 比分记录（可选，支持多盘、三种赛制）
- 球友评级体系（信誉分 + 可信度 + 校准度，聚合为 S/A/B/C，新用户初始 A）
- 真实 ELO 后台计算（驱动撮合，不展示）

_约球_
- 约球发布（底部弹窗，含性别限制、费用分摊）
- 约球列表（全部 + 推荐 Tab）
- 约球详情（访客视角 + 创建人管理视角）
- 审批流（`join_mode=approval`）

_页面_
- 首页「发现约球」
- 约球详情页
- 发布约球弹窗
- 「我的」页面（含我发布的 / 我参与的 / 我的评价 / 我的视频入口）
- 球员主页（公开）
- 评价页面（约球结束后弹窗）
- 管理约球页（创建人视角）
- 我发布的 / 我参与的列表页

**不包含（后续迭代）：**
- UTR 三方接入
- 场地数据库
- 消息通知系统
- 约球内聊天
- 地图找球场功能
- 关注 / 好友体系
- 比分输入交互（待讨论后设计）
