## 一、系统概览

网球约球系统，面向个人网球选手，核心价值是「快速找到合适的球友约球」。

**底部导航：**
- `发现约球`（首页）
- `+`（悬浮按钮，点击弹出发布弹窗）
- `我的`

### 2.3 他人评价

约球结束后，参与者可对同场其他人进行评价，每场约球每对用户只能评价一次。

**评价内容（两部分）：**

① NTRP 投票（三元，必选）

> 「你觉得对方水平比他自评的怎么样？」
> 👆 高了 / 差不多 / 👇 低了

② 个性化标签（多选，选填）

**预设标签（MVP）：**
正手稳、反手稳、发球好、底线稳、网前好、移动快、守时、友善、爽约（负向）

**数据模型：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `review_id` | string | 评价 ID |
| `from_user_id` | string | 评价人 |
| `to_user_id` | string | 被评价人 |
| `booking_id` | string | 关联约球，同一约球只能评价一次 |
| `ntrp_vote` | enum | `higher` / `same` / `lower` |
| `tags` | string[] | 标签列表 |
| `created_at` | datetime | 评价时间 |

### 2.4 用户画像体系

用两个维度对外描述一个球员：**自评 NTRP** + **球友评级**。

```
对外展示    自评 NTRP  +  球友评级（S / A / B / C）
                               ↓ 点击展开
                  信誉分 · 可信度 · 校准度（三项子分）
后台隐藏    真实 ELO 分数（仅用于撮合，不对用户暴露）
```

#### 球友评级（Buddy Rating）

对外只展示等级字母，不展示总分数值。

**总分计算：**

```
总分（0–100）= 信誉分 × 50% + 可信度 × 30% + 校准度 × 20%
```

| 等级 | 分数区间 | 含义 |
|------|---------|------|
| S | 90–100 | 高度可信 |
| A | 75–89 | 可靠球友 |
| B | 55–74 | 基本可信 |
| C | < 55 | 数据不足或存在问题 |

**展示位置：**
- 「我的」页面：球友评级卡片，可点击展开三项子分
- 约球详情页发布者卡片：显示等级字母，可展开查看子分明细
- 约球列表：不展示

---

#### 子维度一：信誉分（权重 50%，满分 100）

默认 100 分，行为驱动扣加分：

| 行为 | 分值变化 |
|------|---------|
| 被标记「准时」 | +2，上限 +10 |
| 连续 5 次准时 | 额外 +5 |
| 被标记「迟到」 | −10 |
| 被标记「爽约」 | −25 |

下限 0，不会为负。准时/迟到标记由约球结束后参与者互评产生。

---

#### 子维度二：可信度（权重 30%，满分 100）

| 来源 | 分值 |
|------|------|
| 近 90 天完成约球（每次 +8，上限 48） | 0–48 |
| UTR 账号绑定 | +25 |
| 上传打球视频（每个 +9，上限 27） | 0–27 |

> 取近 90 天而非累计，防止老用户躺分。

---

#### 子维度三：校准度（权重 20%，满分 100）

基于他人对自评的三元投票（高了 / 差不多 / 低了）：

| 状态 | 分数 | 说明 |
|------|------|------|
| 票数 < 3 | 80 | 数据不足，给中性基准分 |
| 偏差 < 20% | 100 | 自评准确 |
| 偏差 20–50%，偏低（谦虚） | 75 | 实际比自评强，轻微扣分 |
| 偏差 20–50%，偏高（虚报） | 50 | 实际比自评弱，重扣 |
| 偏差 > 50%，偏低 | 55 | 明显谦虚，建议更新自评 |
| 偏差 > 50%，偏高 | 20 | 明显虚报，重扣 |

偏高惩罚重于偏低，因为虚报水平对其他球友伤害更大。

**系统建议：** 偏差强度 > 20% 时，推送「根据 N 位球友反馈，建议将自评从 X 调整为 Y」。

---

#### 真实 ELO（后台计算，不对用户展示）

用于撮合算法，不暴露具体数值。

**数据来源权重（随可信度动态调整）：**

| 数据源 | 可信度低（<30分） | 可信度中（30–80分） | 可信度高（>80分） |
|--------|-----------------|-------------------|-----------------|
| 自评 NTRP | 70% | 30% | 5% |
| 他人投票方向 | 20% | 40% | 35% |
| 战绩（ELO 压力） | 0% | 20% | 40% |
| UTR 换算（如绑定） | 10% | 10% | 20% |

**战绩影响：**
- 赢了自评更高的人 → 真实 ELO 上调压力
- 输给自评更低的人 → 真实 ELO 下调压力
- 比分悬殊系数：6-0 > 6-4 > 7-6，悬殊越大调整幅度越大
- 约球类型权重：竞技（1.0）/ 练习（0.4）/ 陪练（0）

**自评修改限制：**

| 可修改幅度上限 | 冷却期 |
|--------------|--------|
| 可信度 < 30：±1.0 级 | 30 天 |
| 可信度 30–80：±0.5 级 | 60 天 |
| 可信度 > 80：±0.5 级 | 90 天 |

修改后若幅度 ≥ 0.5 级，进入「核查期」：可信度积分冻结（不清零），对其他用户显示「⚠ 近期更新了自评」，完成 2 场约球并收到他人投票后解除。

下调自评的冷却期减半（主动降级视为诚实行为）。

## 三、约球模块

### 3.1 约球列表页（首页）

**布局：** 列表流 + Tab 筛选

| 元素 | 说明 |
|------|------|
| 顶部位置筛选 | 显示当前城市/区域，可点击修改 |
| Tab 栏 | 全部 / 推荐 / 单打 / 双打 |
| 约球卡片 | 见下方字段 |

**约球卡片展示字段：**
- 标题（自动生成或用户填写）
- 时间（日期 + 开始时间）
- 场地名称
- 类型（单打/双打/多人）
- 当前人数 / 上限人数
- 期望水平（NTRP 范围/指定/以上/以下）
- 水平匹配标签（✓ 匹配 / ⚠ 偏高 / ⚠ 偏低）
- 报名模式（直接加入 / 需审核）

**推荐逻辑（MVP）：**
- 与当前用户 NTRP 差值 ≤ 1.0 的约球优先展示
- 同城市优先

### 3.2 发布约球

**交互：** 点击底部 `+` 按钮，底部弹出半屏弹窗

#### 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 否 | 选填；不填时自动生成：`{类型} · {星期} {时间} · {场地简称}` |
| `match_type` | enum | 是 | `single`（单打）/ `double`（双打）/ `multi`（多人） |
| `max_players` | int | 是 | 人数上限，最小 2，最大 10 |
| `date` | date | 是 | 约球日期 |
| `start_time` | time | 是 | 开始时间 |
| `duration` | float | 是 | 时长，滚轮选择：0.5 / 1.0 / 1.5 / 2.0 / 2.5 / 3.0（小时） |
| `venue_name` | string | 是 | 场地名称，来自地图选点 POI |
| `venue_address` | string | 是 | 详细地址 |
| `venue_lat` | float | 是 | 纬度 |
| `venue_lng` | float | 是 | 经度 |
| `level_mode` | enum | 否 | `range`（范围）/ `exact`（指定）/ `above`（以上）/ `below`（以下） |
| `level_min` | float | 否 | NTRP 下限，步长 0.5，范围 1.5~7.0 |
| `level_max` | float | 否 | NTRP 上限，步长 0.5，范围 1.5~7.0 |
| `gender_limit` | enum | 是 | `any`（不限）/ `male`（仅男）/ `female`（仅女），默认 `any` |
| `join_mode` | enum | 是 | `direct`（直接加入）/ `approval`（需审核） |
| `min_credibility` | enum | 是 | `none` / `junior` / `mid` / `senior`，默认 `none` |
| `cost_items` | array | 是 | 费用项列表，见下方结构 |

**cost_items 结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 费用项名称，如「球场费」「球费」 |
| `total_amount` | int | 总费用（分），前端展示元 |

**费用分摊计算：**
`每人分摊 = sum(cost_items.total_amount) ÷ max_players`，人数变化时实时重算，满员后锁定。

**NTRP 选择器交互：**
- 平时显示文本框（如 `3.5 ~ 4.5`）
- 点击数值框触发底部滚轮选择器
- 滚轮可选值：1.5 / 2.0 / 2.5 / 3.0 / 3.5 / 4.0 / 4.5 / 5.0 / 5.5 / 6.0 / 6.5 / 7.0

### 3.3 约球详情页

**页面结构（从上到下）：**

1. **顶部标题区**（蓝色背景）
   - 标题
   - 状态标签：水平匹配状态 / 报名模式 / 招募状态

2. **信息模块**（白色卡片）
   - 2×2 网格：类型 / 人数 / 时间（含时长）/ 水平要求
   - 场地单独一行：场地名 + 详细地址 + 「查看地图 ›」

3. **发布者卡片**
   - 头像 / 昵称 / NTRP / 活跃度星级
   - 「查看主页 ›」入口

4. **参与者卡片**
   - 头像列表（已加入人员）
   - 空位用虚线圆圈占位

5. **AA 费用卡片**
   - 各费用项明细
   - 每人分摊 = 总费用 ÷ 人数

6. **底部操作区（按视角区分）**

#### 访客视角（非创建人）

| 状态 | 按钮 |
|------|------|
| 招募中 + 未报名 + `direct` | 「立即报名」 |
| 招募中 + 未报名 + `approval` | 「申请加入」 |
| 已报名 / 待审核 | 「已报名」（不可点击） |
| 已满员 | 「已满员」（不可点击） |

#### 创建人视角

底部显示「管理约球」按钮，进入管理页，包含：

- **编辑信息**：可修改时间、场地、水平要求、费用等（满员后不可修改人数上限）
- **关闭约球**：手动将状态改为 `closed`
- **审批列表**（仅 `join_mode=approval` 时显示）：
  - 展示所有 `pending` 状态的申请，显示申请人头像 / 昵称 / NTRP / 活跃度
  - 操作：「通过」/ 「拒绝」
  - 通过后 `current_players +1`，达到 `max_players` 时状态自动变 `full`

## 四、「我的」页面

**页面结构（从上到下）：**

1. **顶部个人信息**（蓝色背景）
   - 头像 / 昵称
   - NTRP 自评 + 球友评级字母（一行摘要）
   - 「编辑资料」按钮

2. **快捷入口**（4 格）
   - 我发布的 / 我参与的 / 我的评价 / 我的视频

3. **球友评级**（可展开卡片）
   - 等级字母（S/A/B/C）+ 点击展开信誉分 / 可信度 / 校准度三项子分

4. **球员评价摘要**
   - 自评 NTRP / 校准度状态 / 他评次数（3 格）
   - 他人标签 Top 展示

## 五、功能边界（MVP 范围）

**包含：**
- 手机号注册 + 微信小程序注册
- 球员信息完善（自评 NTRP、视频上传）
- 他人评价（NTRP 三元投票 + 个性化标签）
- 球友评级体系（信誉分 + 可信度 + 校准度，聚合为 S/A/B/C 等级）
- 真实 ELO 后台计算（驱动撮合，不展示）
- 自评修改限制与核查期
- 约球发布（底部弹窗，含性别限制）
- 约球列表（全部 + 推荐 Tab）
- 约球详情（访客视角 + 创建人管理视角）
- 审批流（`join_mode=approval`）
- 「我的」页面

**不包含（后续迭代）：**
- UTR 三方接入
- 场地数据库
- 消息通知系统
- 约球内聊天
- 地图找球场功能
