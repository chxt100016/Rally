# 页面设计文档生成

触发条件：当用户明确要求生成「页面设计文档」或「pagedoc」时触发。

---

## 执行流程

### 1. 确认页面信息

询问用户：
- 页面名称（中文）
- 页面标识（英文，用于文件夹名）
- 页面功能简述

### 2. 创建目录结构

```
docs/plan/Rally/page/{页面标识}/
├── api.md          # API 文档
├── flow.md         # 流程文档
└── sub/            # 子流程/弹窗文档
```

### 3. 生成 api.md

包含以下内容：

```markdown
# {页面名称} · API 文档

## 接口列表

### 1. {接口名称}

```
{METHOD} /api/rally/wechat/{path}
Authorization: Bearer <token>
Content-Type: application/json
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ... | ... | ... | ... |

**响应：** `Result<{DTO}>`

```json
{
  "code": 0,
  "data": {...}
}
```

**{DTO} 字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| ... | ... | ... |

---

## 枚举说明

### {枚举名称}（{字段名}）

| 值 | 说明 |
|----|------|
| ... | ... |

---

## 异常码

| 错误码 | 场景 | 前端处理 |
|--------|------|---------|
| ... | ... | ... |
```

## 注意事项

### 文件引用规则

1. **表格中不要使用 `[[...]]` 文件引用**，会被解析成表格的一部分
2. 需要引用文件时，在表格下方使用列表形式：
   ```
   **{区域}引用：**
   - {功能} → [[sub/{文件名}|{显示名称}]]
   ```

### 接口引用规则

1. 接口引用使用 `[[api#{序号}. {名称}|{METHOD} {path}]]` 格式
2. 示例：`[[api#1. 获取我的档案|GET /wechat/user/profile/me]]`

### 内容规范

1. **不使用绘图**，所有流程使用纯文本描述
2. **不使用 mermaid**，保持文档简洁
3. **枚举 name 使用大写**
4. **DTO 以 DTO 结尾**
5. **Cmd 以 Cmd 结尾**

### 文档结构

1. `api.md` - 接口文档、参数说明、枚举列举、异常码
2. `flow.md` - 页面操作交互、展示字段、触发的弹窗
3. `sub/*.md` - 子流程/弹窗的详细文档
